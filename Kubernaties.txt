Advantages of ingress-nginx:
🧠 ingress-nginx is an actual Ingress Controller implementation using NGINX.
🌐 Ingress is just a Kubernetes resource (a rule/definition), not a controller.
Open-source & flexible	Full control over behavior and config
✅ Built-in Load Balancing	Smart routing to multiple pods
✅ SSL Termination	Easily add HTTPS via cert-manager or static TLS
✅ Path-based & host-based routing	Route traffic based on subdomain or URL paths
✅ Authentication & Rate Limiting	With annotations — protect services quickly
✅ Rewrite/Redirect support	Use rewrite-target, custom status codes, etc.
✅ Modular	Can be used with any cloud (EKS, GKE, AKS, on-prem, kind, etc.)
✅ Supports custom headers	Can inject headers into requests, useful for auth, trace, etc.
✅ Integrates with Prometheus	Full metrics + monitoring ready
✅ Supports Let's Encrypt (cert-manager)	Auto TLS certificates!
