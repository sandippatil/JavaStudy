What is TypeScript:
TypeScript is a programming language developed by Microsoft that builds on JavaScript by adding 
static type checking.

Superset of JavaScript: Any valid JavaScript code is also valid TypeScript code.
Optional static typing: You can define types for variables, function parameters, return values, etc.
Compiles to JavaScript: TypeScript code is compiled (transpiled) to standard JavaScript so it can run in any browser or JavaScript runtime.
Client and server side: 
Object Oriented: 

Why use TypeScript?
✅ Catch errors early: Type errors are caught during development, not at runtime.
🧠 Better IDE support: Features like autocompletion, refactoring, and code navigation are more powerful.
🔄 Maintainability: Helps manage large codebases with clear contracts and fewer bugs.
🤝 Improved team collaboration: Code is more self-documented with types.

let vs const:
| **Feature**                         | **`let`**                                   | **`const`**                                            |
| ----------------------------------- | ------------------------------------------- | ------------------------------------------------------ |
| **Reassignment**                    | ✅ Allowed                                   | ❌ Not allowed (value is constant)                      |
| **Initialization**                  | Can be declared without initializing        | Must be initialized at declaration                     |
| **Block Scope**                     | Yes                                         | Yes                                                    |
| **Mutability (for objects/arrays)** | The variable can be reassigned              | The reference is constant, but **contents** can change |
| **Best Use Case**                   | When the variable needs to be updated later | When the value should never change                     |

let count = 1;
count = 2; // ✅ Works fine

const name = "Alice";
// name = "Bob"; ❌ Error: Cannot assign to 'name' because it is a constant.

const user = { name: "John" };
user.name = "Doe"; // ✅ Allowed: we changed property, not the reference

// user = { name: "Someone else" }; ❌ Error: Cannot reassign the const variable


****
let vs var:

| **Feature**               | **`let`**                                            | **`var`**                                            |
| ------------------------- | ---------------------------------------------------- | ---------------------------------------------------- |
| **Scope**                 | Block-scoped (`{}`)                                  | Function-scoped                                      |
| **Hoisting**              | Hoisted but **not initialized** (temporal dead zone) | Hoisted and **initialized as `undefined`**           |
| **Re-declaration**        | ❌ Not allowed in the same scope                      | ✅ Allowed in the same scope                          |
| **Global Object Binding** | ❌ Not attached to `window` in browsers               | ✅ Attached to `window` in browsers                   |
| **Use in Loops**          | Safer due to block scoping                           | Can cause bugs due to shared scope across iterations |
| **Best Practice**         | ✅ Recommended                                        | ❌ Avoid using `var`                                  |


function testVar() {
    if (true) {
        var a = 10;
    }
    console.log(a); // ✅ 10 — var is function-scoped.. can be used anywhere in function after declaration
}

function testLet() {
    if (true) {
        let b = 20;
    }
    console.log(b); // ❌ Error — b is block-scoped.. allowed only within block.. its like java veriables..
}


console.log(x); // undefined (hoisted)
var x = 5;

console.log(y); // ❌ ReferenceError: Cannot access 'y' before initialization
let y = 10;

var a = 1;
var a = 2; // ✅ redeclaration Allowed

let b = 1;
let b = 2; // ❌ SyntaxError: Identifier 'b' has already been declared

******** Dont use var ********
Always use let or const – they’re predictable and safe.
Avoid var, especially in modern TypeScript or JavaScript projects.

Any type:
The any data type is the super type of all types in TypeScript. It denotes a dynamic type. Using the any
type is equivalent to opting out of type checking for a variable.

Built-in types:
number, string, boolean, void, null, undefined,symbol , object, never

let first_name: string = 'John';
let last_name: string = "Doe";
let full_name: string = `${first_name} ${last_name}`;

Object type:
The object is a non-primitive data type, which can contain any value that is not a number, string, 
boolean, symbol, null, or undefined. You can create an object using either object literal or Object()
constructor.

let person: object = {name: "Bob"};

Array Methods:
| Method          | Description                                     | Example                    |
| --------------- | ----------------------------------------------- | -------------------------- |
| `forEach()`     | Executes a function on each element             | `arr.forEach(x => ...)`    |
| `map()`         | Transforms each element and returns a new array | `arr.map(x => x * 2)`      |
| `filter()`      | Returns elements that match a condition         | `arr.filter(x => x > 5)`   |
| `reduce()`      | Reduces array to a single value                 | `arr.reduce((a,b) => a+b)` |
| `reduceRight()` | Like `reduce()`, but from right                 | `arr.reduceRight(...)`     |
| `some()`        | Returns `true` if any element matches           | `arr.some(x => x > 0)`     |
| `every()`       | Returns `true` if all elements match            | `arr.every(x => x > 0)`    |
| `find()`        | Returns the first matching element              | `arr.find(x => x > 3)`     |
| `findIndex()`   | Returns index of first matching element         | `arr.findIndex(...)`       |

| Method         | Description                             | Example                |
| -------------- | --------------------------------------- | ---------------------- |
| `push()`       | Adds element(s) to the end              | `arr.push(5)`          |
| `pop()`        | Removes last element                    | `arr.pop()`            |
| `shift()`      | Removes first element                   | `arr.shift()`          |
| `unshift()`    | Adds element(s) to the beginning        | `arr.unshift(1)`       |
| `splice()`     | Adds/removes elements at specific index | `arr.splice(1, 1)`     |
| `fill()`       | Fills with a static value               | `arr.fill(0)`          |
| `copyWithin()` | Copies a sequence to another index      | `arr.copyWithin(0, 2)` |

| Method          | Description                 | Example              |
| --------------- | --------------------------- | -------------------- |
| `indexOf()`     | Index of first occurrence   | `arr.indexOf(3)`     |
| `lastIndexOf()` | Index of last occurrence    | `arr.lastIndexOf(3)` |
| `includes()`    | Checks if value is in array | `arr.includes(4)`    |

| Method      | Description          | Example         |
| ----------- | -------------------- | --------------- |
| `sort()`    | Sorts array in-place | `arr.sort()`    |
| `reverse()` | Reverses the array   | `arr.reverse()` |

| Method       | Description                | Example                     |
| ------------ | -------------------------- | --------------------------- |
| `concat()`   | Merges arrays              | `arr.concat([1, 2])`        |
| `slice()`    | Returns a portion of array | `arr.slice(1, 3)`           |
| `join()`     | Joins elements into string | `arr.join(", ")`            |
| `toString()` | Converts array to string   | `arr.toString()`            |
| `flat()`     | Flattens nested arrays     | `[[1], [2]].flat()`         |
| `flatMap()`  | Maps and flattens array    | `arr.flatMap(x => [x * 2])` |
